# Implementation Plan: Cadastro Completo de Usuários

**Branch**: `011-user-registration-form` | **Date**: 2026-04-23 | **Spec**: `specs/011-user-registration-form/spec.md`
**Input**: Feature specification from `specs/011-user-registration-form/spec.md`

## Summary

Expand the existing user registration/edit form from basic fields (name, email, profile, user type, password) into a comprehensive multi-tab form covering: general information (birth date, birthplace, phone, guardians, profile selector with permission-gated editing, user type, scale inclusion), address (reusing the existing polymorphic Address pattern from Church), sacraments (community affiliation and received sacraments), and functions (N:N ministry roles). This feature enriches volunteer profiles to support future scheduling and pastoral management.

## Technical Context

**Language/Version**: Node.js, TypeScript 6.0
**Primary Dependencies**: AdonisJS 7.x (core, auth, bouncer, lucid, inertia, vite), Vue 3.5, PrimeVue 4.5, TailwindCSS 4.2, Inertia.js Vue3 2.3, @vinejs/vine
**Storage**: PostgreSQL (via @adonisjs/lucid), Redis for session/cache
**Testing**: Japa + @japa/api-client + Playwright (e2e)
**Target Platform**: Web (responsive, mobile-first)
**Project Type**: monolith-web-app
**Performance Goals**: Page load < 1s, form save < 500ms
**Constraints**: Soft-delete for all entities, RBAC permission checks on every mutating controller action, CEP lookup service for address auto-fill
**Scale/Scope**: Single parish deployment, < 5k users
