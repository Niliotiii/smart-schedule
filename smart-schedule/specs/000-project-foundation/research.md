['# Research: Project Foundation Setup\n\n## Decisions\n\n### 1. Inertia.js Integration Approach\n**Decision**: Use official `@adonisjs/inertia` package with AdonisJS v6.\n**Rationale**: Provides seamless SSR, shared props, and middleware support. Maintained by AdonisJS team.\n**Alternatives**: Plain SPA with API (rejected because SSR is required for SEO and initial load performance).\n\n### 2. Docker Configuration\n**Decision**: Single container for AdonisJS app + separate PostgreSQL container.\n**Rationale**: Simplicity for development, aligns with docker-compose standard. Volumes for code and DB persistence.\n**Alternatives**: Multi-container with separate frontend build (rejected because monólito simplifies HMR).\n\n### 3. TailwindCSS Setup\n**Decision**: Install TailwindCSS via npm, configure with PostCSS and Vite plugin.\n**Rationale**: Standard approach, works well with Inertia and Vite dev server.\n**Alternatives**: CDN version (rejected because not suitable for production builds).\n\n### 4. Database Migration Strategy\n**Decision**: Use AdonisJS Lucid migrations with timestamps.\n**Rationale**: Built-in, version-controlled, easy rollback.\n**Alternatives**: Raw SQL scripts (rejected because loses ORM benefits).\n\n### 5. Testing Stack\n**Decision**: Vitest for frontend unit tests, Jest/Maya for backend (to be decided in /speckit.tasks).\n**Rationale**: Vitest integrates with Vite, fast. Backend tests can use same runner or different.\n**Alternatives**: Separate Cypress/E2E (deferred to later phase).']
